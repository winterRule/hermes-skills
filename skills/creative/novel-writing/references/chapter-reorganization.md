# 章节重组工作流（大纲重编号后批量重映射）

当大纲从v2/v3升级到v4（例如250章→800章扩写）后，现有章节文件需要按新编号重新组织。本文件记录已验证的批量重组模式。

## 前置条件
- 新大纲（v4.0终极版）已生成，每章有明确编号、标题和内容要点
- 现有章节文件使用旧编号（但连续、无缺失）
- 新旧编号映射基于**内容**而非编号（旧标题与新大纲标题匹配）

## 步骤

### 1. 扫描现有文件
```python
import os, re
base = '/mnt/d/sideline/ai/novel/中土纪元'
old_files = {}
for vol in ['第一卷','第二卷','第三卷','第四卷','第五卷']:
    vp = os.path.join(base, vol)
    if os.path.isdir(vp):
        for fn in sorted(os.listdir(vp)):
            m = re.match(r'第(\d+)章', fn)
            if m:
                old_files[int(m.group(1))] = os.path.join(vp, fn)
```

### 2. 建立内容对齐映射
不要假设旧编号=新编号。必须按章节标题与大纲的一一对应来确定映射。常见情况：旧011章「江南水乡」实际对应新021章（第二弧线），而非新011章（第一弧线扩张）。

映射表应逐段确定：
```
旧001-010 → 新001-010 (第一弧线，不变)
旧011-020 → 新021-030 (第二弧线，偏移+10)
旧021-030 → 新041-050 (第三弧线，偏移+20)
旧031-039 → 新066-074 (第四弧线，偏移+35)
...
```

### 3. 处理旧→新1:N映射
当旧章被分配到更大范围的新编号时（如旧061-070的10章对应新151-180的30章），使用均匀间隔插入：
```python
# 每3章取1，插入到新编号中
targets = [151, 154, 157, 160, 163, 166, 169, 172, 175, 178]
for i, t in enumerate(targets):
    mapping[61+i] = t
```

### 4. 批量复制+重命名
```python
import shutil
new_base = os.path.join(base, '_v4重组')
for old_num in sorted(mapping.keys()):
    new_num = mapping[old_num]
    new_vol = get_new_vol(new_num)  # 按新卷范围判断
    # 保留原标题，只改编号
    title_part = re.sub(r'^第\d+章_?', '', old_basename)
    new_fname = f'第{new_num:03d}章_{title_part}'
    shutil.copy2(old_path, os.path.join(new_base, new_vol, new_fname))
```

### 5. 验证关键映射
选择各卷代表性的旧章，验证新编号和卷归属正确：
```
旧001「陌生海岸」 → 新001 第一卷 ✅
旧011「江南水乡」 → 新021 第一卷 ✅  (不是新011!)
旧051「北岭陷落」 → 新121 第一卷 ✅
旧121「战前集结」 → 新301 第三卷 ✅
```

### 6. 替换旧目录
验证通过后：
```bash
rm -rf 第一卷 第二卷 第三卷 第四卷 第五卷
mv _v4重组/第一卷_星火初燃 .
# ... 其余卷同理
rmdir _v4重组
```

## 关键陷阱
- **不能假设旧编号=新编号**：扩写后的章节间距不均匀，必须按内容对齐
- **弧线边界可能跨卷**：如第七弧线151-180章的前10章在第一卷，后20章在第二卷
- **备份优先**：始终先复制到临时目录（_v4重组），验证后再删除旧文件
- **保留日志**：生成逐条映射日志文件，方便追溯和调试
