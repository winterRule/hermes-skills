# 名字/年号批量统一技术

## 问题场景
小说大纲设定了架空皇帝名「朱景桓」、年号「景武」，但实际撰写的章节使用了真实历史名「朱厚照」、年号「正德」。在全书完成92.5%时发现468处不一致，涉及约200个文件。

## 诊断命令

```bash
# 1. 从大纲提取标准名
grep "推荐架空名\|年号：" 全书大纲_完整版.txt

# 2. 统计错误名出现次数（排除大纲）
grep -r "朱厚照" --include="*.txt" . | grep -v "全书大纲" | wc -l

# 3. 按文件统计分布
grep -rl "朱厚照" --include="*.txt" . | grep -v "全书大纲"

# 4. 统计正确名出现次数
grep -r "朱景桓" --include="*.txt" . | grep -v "全书大纲" | wc -l
```

## 批量替换命令

```bash
cd <novel_root>

# 第一轮：替换皇帝名（排除大纲文件）
find . -name "*.txt" ! -name "全书大纲*" -exec sed -i 's/朱厚照/朱景桓/g' {} \;

# 第二轮：替换年号
find . -name "*.txt" ! -name "全书大纲*" -exec sed -i 's/正德/景武/g' {} \;
```

## 验证命令

```bash
# 确认错误名已清零
grep -r "朱厚照" --include="*.txt" . | grep -v "全书大纲" | wc -l  # 应为 0
grep -r "正德" --include="*.txt" . | grep -v "全书大纲" | wc -l    # 应为 0

# 确认大纲未被修改（应保留原始设定说明）
grep "朱厚照\|正德" 全书大纲_完整版.txt

# 抽查2-3个文件确认替换后文字通顺
head -20 第三卷/第250章*.txt
```

## 必须保护的内容

以下历史年号是真实历史对照，**不可替换**：
- 弘治（前任皇帝年号，作为历史背景）
- 万历、嘉靖、崇祯（未来历史对照）
- 明代真实人物名（张居正、王守仁等）

判断标准：如果该名称出现在「现代线的主角查阅历史资料」或「对过去/未来历史的叙述」中 → 保留；如果该名称出现在「当前朝代的叙述」中 → 替换。

## 避免的陷阱

1. **不要用全局 `sed` 处理大纲文件** — 大纲是设定来源，应保留原始说明
2. **替换后必须验证** — `grep -c` 确认残留为零
3. **抽查读感** — 替换后的文字必须通顺自然
4. **先查再写** — 写新章节前先做命名一致性检查，不要扩散错误
