# 字数验证方法对比

本文档记录在小说创作session中实际可用的三种字数验证方法及其精度差异。

## 方法对比

### 方法A：`wc -c`（技能推荐）★★★★★
```bash
wc -c <章节文件>
```
- **优点**：最快、最可靠、直接给出字节数
- **缺点**：terminal工具可能超时（WSL环境下大文件）
- **阈值**：<6000字节 → 必然不够2000字；6000-7000 → 边界；>7000 → 安全

### 方法B：Python `len(re.findall(r'[\u4e00-\u9fff]', text))`（AI常用）★★★
```python
from hermes_tools import read_file
import re
data = read_file('章节路径')
cn = len(re.findall(r'[\u4e00-\u9fff]', data['content']))
```
- **优点**：execute_code可用，不依赖terminal
- **⚠️ 严重缺陷**：只统计[\u4e00-\u9fff]范围的汉字，**排除中文标点**（，。！？：；""''等在全角符号范围U+FF00-U+FFEF，不在汉字范围）
- **后果**：一个1969汉字的章节加上标点实际可能是2100+字符，但因为regex只计1969会被误判为"不足2000"
- **校准**：此方法计数值 × 1.08 ~ 1.12 ≈ 实际总字符数（含标点）

### 方法C：`len(re.sub(r'\s', '', text))`（含标点）★★★★
```python
total = len(re.sub(r'\s', '', text))  # 去掉空白后全部字符
```
- **优点**：包含中英文标点，更接近真实"字数"
- **缺点**：包含了卷标和标题行（约30字符），需减去
- **阈值**：total - 30 > 2050 → 安全（含标点后2000+）

## 本次session的教训

第076章和第077章被方法B判断为"不足2000字"（1969和1982），但实际含标点应为2100+和2150+。虽然最终通过patch补到了2000+，但这造成了不必要的重工。

**教训**：当方法B显示1950-2000时，先用方法C复核。如果方法C显示2100+，大概率不需要补。

## 推荐验证流程

1. **首选**：`wc -c`，阈值6000字节（对应~2000字含标点）
2. **备选（terminal不可用时）**：先用方法C（含标点总数），阈值2050
3. **方法B仅作参考**：1950以下肯定不够，1950-2000需要方法C交叉确认
