# 大纲重建与长篇扩写规划

## 大纲编号错位检测与修复

### 问题识别
初始大纲的章节编号（如"第二卷201–400章"）与实际创作的文件编号（如"第二卷051–120章"）可能完全错位。原因：大纲在创作前编写，实际创作中未按大纲编号执行；大纲从未被更新以反映实际文件结构。

### 检测方法
用 `execute_code` 扫描所有卷目录，提取实际文件名中的章号和弧线结束标记：

```python
import os, re

base = "/mnt/d/sideline/ai/novel/{书名}/"
for vol in ["第一卷", "第二卷", "第三卷", "第四卷", "第五卷"]:
    files = sorted(os.listdir(os.path.join(base, vol)))
    ch_nums = sorted([int(f[1:4]) for f in files if f.endswith('.txt')])
    print(f"{vol}: {len(files)}章, {ch_nums[0]}-{ch_nums[-1]}")
```

同时提取弧线边界：
```python
for f in files:
    text = read file
    m = re.search(r'第[二三四五六七八九十百]+阶段「[^」]+」终', text)
    if m: record arc boundary
```

### 重建流程（v2.0 → v3.0）

1. **扫描实际结构**：运行上述脚本得到真实章号分布和弧线边界
2. **对比原大纲**：标记所有编号不一致的弧线和章节
3. **确认连续性**：检查章号是否连续（无缺章）—— `for i in range(min, max+1): if i not in ch_nums: print(f"缺{i}")`
4. **重写大纲**：以实际编号为准，保留原大纲的主题描述，更新所有章节编号
5. **用户确认**：呈递新旧对比，等用户确认后继续

### 本会话案例
- 原大纲：Vol 2 = 201–400章，Vol 3 = 401–600章
- 实际：Vol 2 = 051–120章，Vol 3 = 121–160章
- 修复：完全重建为v2.0（对齐250章），再扩建为v3.0（800章目标）

## 长篇扩写规划（当前→目标）

### 规模计算
- 当前字数 ÷ 章均字数 = 当前章数
- 目标字数 ÷ 章均字数 = 目标章数
- 目标章数 - 当前章数 = 需新增章数

### 卷分配原则
保持5卷结构，按历史对应阶段的比例分配章数：
- Vol 1（清末至抗战前）：约20%总章数
- Vol 2（抗战全面至统一战线）：约22%
- Vol 3（抗战相持）：约20%
- Vol 4（解放战争）：约22%
- Vol 5（建国后）：约16%

### 现有章节重新编号
使用简单偏移映射：`新编号 = 旧编号 + 偏移量`
- Vol 2: offset = +110（051→161, 120→230）
- Vol 3: offset = +220（121→341, 160→380）
- Vol 4: offset = +340（161→501, 230→570）
- Vol 5: offset = +450（231→681, 250→700）

### 插入章节命名
为避免与现有文件冲突，新插入的章节使用 `B` 后缀：
- `第011B章_逃亡之路.txt`（插入于010和021之间）
- `第031B章_江南的暗面.txt`（插入于030和041之间）

终稿时统一重编号。

### 扩写要点
- 每个战役/事件从1章扩展到3-5章
- 添加过渡章节连接现有章节
- 扩展角色背景故事和侧面视角
- 增加民间生活细节章节
